# 统计页面重设计

## 目标

优化统计页面的信息架构与视觉表现，解决月度数据位置误导问题，删除低价值的训练记录列表，采用扁平极简风格重新设计。

## 信息架构

页面从上到下：

1. **页面标题** — "统计"
2. **年度汇总** — 两个色块卡片并排：全年训练次数 | 全年总时长
3. **月份导航** — ← 2026年3月 →
4. **月度汇总** — 两个色块卡片并排：当月训练次数 | 当月总时长
5. **日历** — 圆形格子热力图

### 数据联动

- 月份导航切换月份时，月度汇总和日历跟随更新
- 年度汇总跟随月份所在年份自动变化（无独立年份切换控件）
- 例：从 2026年1月 切到 2025年12月，年度汇总自动显示 2025 年数据

### 删除

- 训练记录列表（不实用）

## 视觉风格

扁平极简，参考 Nike Training Club 风格：大色块、无阴影、强对比。

### 规则

- **无 border / box-shadow** — 用色块区分层次，不靠描边和阴影
- **汇总卡片** — 品牌色大色块背景 + 白色粗体数字
  - 年度卡片：运动色系（`--exercise-start`）
  - 月度卡片：休息色系（`--rest-start`）
- **日历格子** — 改为圆形
  - 训练日：品牌实心圆填充（`--exercise-start`）
  - 未训练日：浅灰底圆
  - 今天：特殊标识（如描边圆环）
- **间距加大** — 各区域间留更多呼吸空间
- **字体层级拉大** — 标题 32px bold，汇总数字 36px，标签文字 13px，日历数字 14px

### 主题适配

- 暗色模式和浅色模式使用各自的 CSS 变量，色块背景自动跟随主题切换

## 涉及文件

- `src/views/StatsView.vue` — 模板重构 + 样式重写
- `src/stores/records.ts` — 添加年度统计方法（`yearStats`）

## 不涉及

- 不添加新功能（streak、热力图分级等）
- 不修改数据模型
- 不修改其他页面
